# 11 — Claim Boundary, Trust, Security, and Governance

The project is strongest when it refuses overclaiming.

## Public-alpha boundary

```text
Public-alpha proof architecture.
Not achieved AGI.
Not achieved ASI.
Not empirical SOTA.
Not production authorization.
Not legal approval.
Not user-fund authorization.
Not mainnet activation.
Not guaranteed outcomes.
```

## Trust posture

The trust posture is evidence-first:

```text
No Evidence Docket, no strong public claim.
No eval, no propagation.
No rollback, no release.
Public proof, private data.
```

## Security posture

Interactive demos must remain browser-local unless explicitly authorized:

```text
no live model calls
no live wallets
no transactions
no external systems contacted
no secrets printed
no private customer data
no production activation
```

## Governance posture

GoalOS governance is built around:

```text
Mission Contracts
Claims Matrices
Evidence Dockets
Verifier Reports
Risk Ledgers
Claim Boundaries
Human Authority
Rollback Conditions
Chronicle Entries
Capability Packages
```

## Why caution increases credibility

The strongest version of the project is not:

```text
We achieved AGI.
```

It is:

```text
We are building the proof infrastructure that lets autonomous AI work become institutionally governable.
```

## Falsification discipline

A serious proof system must publish:

```text
negative controls
failed runs
rejected candidates
rollback events
baseline losses
external replay failures
claim boundary decisions
```

A system that hides failure cannot be a proof system.
