
## Objective

Run public QA on the interactive Proof Console and Frontier Release Room.

## Checks

- Browser-local only.
- No external model calls.
- No wallet or transaction paths.
- Keyboard accessibility.
- Mobile layout.
- Downloaded docket validity.
- Clear human-review boundary.

## Acceptance criteria

- QA findings logged.
- Critical usability issues fixed.
- Console remains claim-bounded.
