# 05 — Proof OS and Evidence Dockets

The Proof OS is the practical heart of the project.

A normal AI system produces an answer. GoalOS produces a proof-bearing decision package.

## Mission OS product promise

```text
Set the objective.
GoalOS runs until proof is done.
```

The deliverable is not a document. The deliverable is a **Governed Decision State**.

## Governed Decision State

A Governed Decision State contains:

```text
objective
claims matrix
source provenance
contradiction register
uncertainty ledger
Evidence Docket
verifier report
risk ledger
claim boundary
executive brief
action graph
rollback conditions
Chronicle entry
capability package
```

## Evidence Docket

The Evidence Docket is the proof room.

It should contain:

```text
manifest
claims matrix
environment
baselines
proof packets
evaluator attestations
selection certificate
risk ledger
cost ledger
safety ledger
replay / audit path
public report
private appendix index
```

## Why Evidence Dockets matter

A proof page is not a marketing page. It should show:

```text
what was tested
what passed
what failed
which baselines were used
which gates were enforced
what remains uncertain
how to rerun or audit the claim
```

## Proof debt

Proof debt is the risk that unverified AI output becomes institutional default.

GoalOS reduces proof debt by forcing claims to become explicit, source-bound, risk-classed, and reviewable before action.

## Immediate proof target

```text
Proof Run 001
```

The next decisive milestone is a public-safe mission run that reaches DONE=true and publishes an inspectable Evidence Docket.
