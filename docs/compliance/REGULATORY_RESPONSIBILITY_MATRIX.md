# Regulatory Responsibility Matrix

| Area | GoalOS public-alpha posture | External-party responsibility |
|---|---|---|
| Privacy / GDPR / data protection | No user data wanted; no public data processing intended beyond GitHub platform interactions. | Do not submit personal data. Determine controller/processor roles for any separate pilot. |
| Securities / investment / token | No offer, no sale, no solicitation, no investment advice, no profit promise. | Perform independent securities, commodities, AML/KYC, sanctions, tax, and consumer-law analysis. |
| AI / model governance | Reference architecture and demos only. Human review required. | Determine applicable AI, sectoral, procurement, export, safety, and governance obligations. |
| Cybersecurity | Defensive, claim-bounded, no unauthorized scanning or exploit use. | Use only on systems you own or are authorized to test; follow responsible disclosure and law. |
| IP / content rights | Public-safe docs/code/examples. | Submit only content you own or are licensed/authorized to share. |
| Professional advice | No legal, financial, medical, tax, security, or regulatory advice. | Obtain independent professional advice before reliance or deployment. |
