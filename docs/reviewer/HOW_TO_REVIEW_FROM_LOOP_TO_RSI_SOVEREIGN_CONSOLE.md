# How to Review the From Loop to RSI Sovereign Console

Use the public page as a review surface, not as a production authorization system.

## Reviewer checklist

1. Run the RSI cycle.
2. Stress a Move-37 candidate.
3. Simulate OMNI overreach and confirm it is rejected.
4. Toggle schema-bound artifacts off and confirm schema drift blocks.
5. Toggle public/private proof boundary off and confirm the boundary blocks.
6. Package the dossier and confirm decision state updates.
7. Download every artifact.
8. Confirm the artifacts contain claim boundaries and do not include private data.

## What would falsify the demo claim?

- The page requires a network call.
- The page asks for wallet or user funds.
- OMNI can grant outcome authority.
- Simulated evidence inflates confidence into promotion.
- High novelty bypasses baselines or persistence.
- Downloads omit claim boundaries.

## Boundary

No user data. No user funds. No wallet. No transaction. No network call. No production authority. Human review required.
