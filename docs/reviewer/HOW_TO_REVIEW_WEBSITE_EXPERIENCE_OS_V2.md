# How to review Website Experience OS V2

1. Open `public/index.html`.
2. Confirm the top navigation exposes Start, Atlas, Proof Ledger, Proof Run 001, Registry, Reviewer Room, and Search.
3. Press `/` and verify the command palette opens.
4. Search for `validator`, `docket`, `mission`, `action`, `token`, and `privacy`.
5. Open `website-operating-system.html`, `pathfinder.html`, `demo-ecosystem-registry.html`, and `site-map.html`.
6. Confirm no user data, no user funds, no wallet, no transaction, no network call, no production authority, and human review required are visible.
7. Confirm existing public demo pages are still reachable through the registry or site map.

A reviewer may accept, reject, revise, or dissent. Missing routes should be treated as remediation items, not hidden.
