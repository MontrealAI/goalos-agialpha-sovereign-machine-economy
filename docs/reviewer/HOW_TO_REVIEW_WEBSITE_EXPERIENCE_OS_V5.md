# How to Review Website Experience OS V5

1. Open `public/index.html`.
2. Confirm the proof console is legible and useful.
3. Press `/` and confirm the command palette opens.
4. Open `public/demo-ecosystem-registry.html` and verify the registry is specific, not generic.
5. Open `public/site-map.html` and verify preserved pages remain discoverable.
6. Open `public/token-boundary.html` and verify the token boundary is explicit.
7. Open `public/site-health.html` and verify route QA.
8. Inspect `reports/website-experience-os-v5-qa.json`.

Boundary: No user data. No user funds. No wallet. No transaction. No network call. No production authority. Human review required.
