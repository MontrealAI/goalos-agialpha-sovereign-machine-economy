# How to Review the Real-Task Benchmark Bridge

Use this page to inspect whether a GoalOS claim is ready to become a real benchmark docket.

Reviewer checklist:

1. Is the task family public-safe?
2. Is there a real task manifest?
3. Are B0-B6 baselines included?
4. Are model/tool/budget constraints equalized?
5. Is the ProofBundle / Evidence Docket complete?
6. Is the replay path explicit?
7. Are cost and risk ledgers present?
8. Is there a validator report?
9. Is there a delayed-outcome plan?
10. Is there an independent reviewer path?

If any answer is no, the correct verdict is **hold**, **revise**, or **reject**.
