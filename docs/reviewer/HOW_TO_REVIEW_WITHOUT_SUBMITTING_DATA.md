# How to Review GoalOS Without Submitting User Data

1. Use public pages and public repository files only.
2. Use the sample dockets and generated reports.
3. Run local demos with synthetic or public-safe sample data only.
4. Do not paste personal, customer, confidential, regulated, credential, or proprietary data into issues, PRs, demos, prompts, or dockets.
5. File accept/reject/revise feedback by referencing public artifact paths and public-safe observations.
