# How to Use the Proof Mission Control Board

1. Open `/proof-mission-control.html`.
2. Pick a public-safe mission preset.
3. Run the mission readiness board.
4. Toggle one required artifact off to see how the decision state changes.
5. Download the mission docket.
6. Download the validator packet.
7. Use the GitHub issue draft only if it contains no user data, no funds, no credentials, and no confidential information.

## Reviewer checklist

- Is the objective bounded?
- Are claims separated from evidence?
- Is replay possible?
- Is a validator packet present?
- Are cost/risk ledgers named?
- Is the public/private boundary visible?
- Is human review required?
