# Proof Velocity Metrics

## Purpose

Proof velocity measures how fast the system converts objectives into validated, reusable capability.

## Core metrics

- `missions_started`
- `missions_completed`
- `evidence_dockets_created`
- `validator_reports_created`
- `release_gates_passed`
- `capability_packages_created`
- `capability_reuse_events`
- `median_hours_to_docket`
- `median_hours_to_validation`
- `evidence_completeness_score`

## Interpretation

High proof velocity is not raw activity. It requires bounded missions, complete evidence, validator review, and reuse.


## Claim discipline

This document is an operating standard. It does not claim financing, market valuation, production authorization, legal approval, user-fund authorization, revenue, enterprise adoption, or independent audit completion. Counted evidence must be dated, source-verifiable, and stored in the evidence room.
