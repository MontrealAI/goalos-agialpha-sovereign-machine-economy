# Benchmark and Baseline Protocol

## Purpose

A useful benchmark compares GoalOS against a named baseline on bounded work. The protocol requires replay data and cost/risk accounting.

## Minimum benchmark packet

- Task description.
- Baseline method.
- GoalOS method.
- Metrics.
- Dataset or prompt set.
- Tool traces.
- Cost ledger.
- Risk ledger.
- Reproducibility notes.
- Validator report.

## Recommended metrics

- Completion rate.
- Evidence completeness.
- Validation pass rate.
- Human review burden.
- Time to approved docket.
- Cost per accepted result.
- Reuse rate.


## Claim discipline

This document is an operating standard. It does not claim financing, market valuation, production authorization, legal approval, user-fund authorization, revenue, enterprise adoption, or independent audit completion. Counted evidence must be dated, source-verifiable, and stored in the evidence room.
