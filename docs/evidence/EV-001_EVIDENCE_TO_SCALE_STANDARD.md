# EV-001 — Evidence-to-Scale Standard

## Purpose

GoalOS needs more than a compelling repository. It needs a disciplined way to prove whether autonomous AI work is becoming institutionally valuable. This standard defines the evidence required before strong public scale claims are allowed.

## Core principle

A strategic claim is not accepted because the language is impressive. It is accepted only when the evidence packet contains dated artifacts, clear source provenance, reviewer notes, and reproducible metrics.

## Evidence classes

| Class | What it proves | Counted by default |
|---|---|---:|
| Repository proof | The system, docs, schemas, site, and automation exist. | Yes |
| Operator proof | The system can run missions and generate dockets. | Yes, with logs |
| Pilot proof | Real users or organizations completed bounded pilots. | Only with artifacts |
| Commercial proof | Signed, dated commercial signals exist. | Only with source records |
| Usage proof | Real usage is measured over time. | Only with metric provenance |
| Benchmark proof | Outcomes beat named baselines on bounded tasks. | Only with replay data |
| Independent proof | An external reviewer validates the evidence packet. | Only with reviewer identity and report |
| Fair-value support | A qualified external process supports a private-market value. | Only with formal documentation |

## Evidence statuses

`example_placeholder` means the file is a template or illustrative sample. It never counts as market proof.

`operator_attested` means the maintainer or operator attests that the artifact is real, dated, and internally reviewed. It may count as internal proof.

`source_verifiable` means the artifact includes a source link, signed record, contract metadata, verifiable log, or third-party system of record.

`third_party_verified` means an independent reviewer or counterparty has validated the artifact.

## Strong claim gate

A strong market-scale claim requires verified evidence across product, usage, customer, benchmark, commercial, governance, independent review, and fair-value-support categories. The validator blocks the claim until those categories are present.


## Claim discipline

This document is an operating standard. It does not claim financing, market valuation, production authorization, legal approval, user-fund authorization, revenue, enterprise adoption, or independent audit completion. Counted evidence must be dated, source-verifiable, and stored in the evidence room.
