# Founding Proof Mission Intake

A founding proof mission should be small enough to finish, important enough to matter, public-safe enough to publish, and bounded enough to validate.

Recommended first mission types:

- Repository launch readiness.
- Website-code parity review.
- Frontier release governance review.
- Public demo series review.
- Proof Run 001 readiness.

Private, regulated, customer-data, token/funds, wallet, payment, medical, legal, financial, security-sensitive, or confidential work requires a separate written agreement and independent review.
