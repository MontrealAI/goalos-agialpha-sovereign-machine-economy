# Institutional Pilot Program

This is a practical path for evaluating GoalOS without pretending the system is production-certified.

## Phase 1 — Repository launch

- create the GitHub repository;
- run the autonomous workflow;
- publish the generated website;
- review the claim boundary;
- inspect proof artifacts and QA reports.

## Phase 2 — Single controlled mission

- choose one low-risk internal objective;
- write a Mission Contract;
- produce a ProofBundle and Evidence Docket;
- run at least one validator review;
- record the decision state.

## Phase 3 — Repeatability

- run three similar missions;
- compare cost, time, risk, and validator findings;
- identify reusable capability packages;
- retire anything that does not reproduce.

## Phase 4 — Governance expansion

- add role approvals;
- add release gates;
- add data handling controls;
- add independent review for stronger claims.
