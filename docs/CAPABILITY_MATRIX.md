# Capability Matrix

This matrix turns the grand architecture into practical operator capabilities.

| Capability | Current scaffold | Production path |
|---|---|---|
| Mission scoping | Example Mission Contract and schema | UI intake form, policy checks, versioned templates |
| Job execution | Proof Job schema and examples | orchestrator integration, job queue, permissioned tool use |
| Evidence capture | ProofBundle and Evidence Docket schemas | automatic provenance capture, source pinning, artifact hashing |
| Validation | Validator Report schema | reviewer workflows, multi-validator quorum, delayed outcome checks |
| Governance | Governance docs and vote schema | policy registry, role-based approvals, emergency pause |
| Release | Release Gate schema and protocol docs | release environments, rollback automation, audit export |
| Website publication | generated Pages site and QA | content approvals, public/private redaction, trust center |
| Capability reuse | Chronicle and Capability Package schemas | registry, eval harness, reuse scoring, retirement policy |
| Settlement | Settlement Receipt schema | accounting adapter, non-financial credits, later on-chain adapter after approvals |
| Enterprise trust | Security, claims, threat model | procurement packet, evidence export, vendor risk workflow |

The repository’s value comes from making each capability inspectable before it becomes operationally powerful.
