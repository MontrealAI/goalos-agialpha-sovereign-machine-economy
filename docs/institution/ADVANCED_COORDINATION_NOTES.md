# Advanced Coordination Notes — Multi-Agent Institution Lab

The page implements an illustrative browser-local model of proof-gated coordination.

The advanced panel exposes:

- proof strictness
- validator quorum
- agent count
- budget discipline
- proof controls
- stress mode

The transparent baseline is a teaching/control rail, not an empirical SOTA claim. Learned routers must beat transparent baselines under equal budget, real tasks, replayable proof, and zero critical safety violations before any stronger claim can be promoted.

The page reflects the operating doctrine:

```text
Maximum coordination ≠ maximum autonomy.
Maximum coordination = maximum verified useful work per unit of risk, cost, latency, and proof debt.
```
