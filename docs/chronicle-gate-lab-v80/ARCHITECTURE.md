# Chronicle Gate Architecture

The Chronicle Gate is the improvement-loop firewall.

```text
LLM draft
→ proof debt
→ AGI Jobs
→ ProofBundles
→ Evidence Docket
→ Capability Promotion Gate
→ Chronicle candidate
→ bounded future-mission influence
```

The gate blocks promotion unless all required proof signals are present:

1. ProofBundle exists.
2. `replay_result` exists and passes.
3. `validator_verdict` exists and approves.
4. Claim boundary is preserved.
5. Human-review hold and rollback metadata are recorded.

The point is simple: attractive output can create work, but it cannot become institutional memory without proof.
