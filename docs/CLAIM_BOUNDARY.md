# Claims Policy

This repository uses a simple public trust rule:

> Grand vision is allowed. Unsupported empirical claims are not.

## Allowed language

You may describe this repository as:

- a proof-settled AI work scaffold;
- a GoalOS-native architecture;
- a research prototype;
- an evidence-first operating surface;
- a design for governed intelligence organizations;
- a public repository kit;
- a non-custodial demonstration;
- a claim-bounded implementation scaffold.

## Restricted language

Do not claim, imply, or market this repository as:

- achieved AGI;
- achieved ASI;
- superintelligence;
- empirically state-of-the-art;
- production certified;
- externally audited, unless the audit exists and is linked;
- legally approved;
- tax approved;
- financial advice;
- guaranteed secure;
- guaranteed profitable;
- mainnet activated, unless the relevant release-state evidence exists;
- authorized for user funds;
- autonomous legal sovereignty;
- civilization-scale capability.

## Evidence upgrade path

A stronger claim needs an Evidence Docket containing:

1. mission contract;
2. source cards;
3. job packet;
4. tool traces;
5. output hash;
6. replay or reproduction notes;
7. validator report;
8. contradiction register;
9. risk ledger;
10. cost ledger;
11. delayed outcome check when relevant;
12. settlement receipt or rejection note.

No docket, no strong claim.

## Website publication rule

Every generated public page must contain or link to this claim boundary.
