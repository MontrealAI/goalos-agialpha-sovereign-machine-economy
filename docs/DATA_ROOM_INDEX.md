# Institutional Data Room Index

Use this index when presenting the repository to serious partners, reviewers, or collaborators.

| Folder | Why it matters |
|---|---|
| `CLAIMS.md` | shows claim discipline |
| `SECURITY.md` | gives vulnerability reporting posture |
| `GOVERNANCE.md` | shows how decisions are controlled |
| `docs/` | gives architecture and operator context |
| `standards/` | defines artifact laws |
| `schemas/` | provides machine-readable contracts |
| `examples/` | demonstrates the proof chain |
| `scripts/` | shows QA, site generation, digesting, and validation |
| `reports/` | records generated assurance outputs |
| `public/` | exposes the generated website |

A serious repository should let a reviewer understand what exists, what is planned, what is explicitly not claimed, and what evidence supports each public statement.
