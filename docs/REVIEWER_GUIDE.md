# Reviewer Guide

Purpose: inspect claims and evidence. Audience: external reviewers. What to do next: open Proof Run 001, claims matrix, replay path, validator reports, cost/risk ledgers, and boundary docs. Related reports/scripts: `validate_claims.py`, `verify_site.py`.

Reviewer honesty box: What would change your mind? What would falsify the claim? Which evidence is missing? Which claim is too broad? Which baseline should be stronger? Which gate should block promotion? Which download or replay path failed?

Accept, reject, revise, or dissent only after checking Evidence Docket, ProofBundles/proof packets, baselines, replay path, cost/risk ledger, public/private boundary, and validator reports.
