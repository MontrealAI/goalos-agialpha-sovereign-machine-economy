# Reviewer guide

## Inspect

1. Open the docket. 2. Read the claim. 3. Check the claims matrix. 4. Compare baselines. 5. Inspect proof packets and validator reports. 6. Run replay steps. 7. Review cost/risk and rollback posture. 8. Record accept, reject, revise, or dissent.

## Sufficient evidence

Public-safe artifact paths, reproducible commands, bounded claims, validator notes, baselines, and explicit limitations.

## Not evidence

Marketing language, screenshots without provenance, private assertions, unavailable datasets, unsupported claims, or token/market statements.
