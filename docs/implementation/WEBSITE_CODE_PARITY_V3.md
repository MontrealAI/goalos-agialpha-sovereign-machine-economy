# GoalOS Website-to-Code Parity V3

This standard requires every major public website capability to have at least one repository-backed implementation artifact: code, test, report, docket, documentation, or executable script.

The public website may be cinematic. The repository must remain inspectable.

## Parity rule

A website capability is considered covered only when:

1. the public page exists;
2. a matching implementation artifact exists;
3. a test, report, issue body, or generated docket exists;
4. the claim boundary is preserved;
5. no external authority, wallet, mainnet settlement, user-fund authorization, or production activation is granted.

AI creates output. GoalOS creates proof.

No Evidence Docket, no strong claim.
No eval, no propagation.
No rollback, no release.
Human authority remains required.
