# GoalOS Implementation Parity Standard

Public website capabilities must have repository implementation artifacts. A capability is considered covered only when it has at least one code artifact, one test or generated report, and a claim boundary.

The standard intentionally keeps the implementation local and public-alpha. It does not authorize production operation, wallet use, mainnet settlement, external actuation, or autonomous legal authority.

## Required public-alpha capability families

- Mission Contracts
- Proof Jobs
- Tool Traces
- Evidence Dockets
- Validator Reports
- Governed Decision States
- Settlement Receipts, simulation-only
- Chronicle Memory
- Capability Packages
- AEP-001 Object Model
- Evidence Docket 6.1
- Proof Gradient
- Mission OS until-DONE loop
- Frontier Release Room
- Multi-Agent Institution
- Proof Run 001

## Boundary

No proof, no settlement. No Evidence Docket, no strong claim. No eval, no propagation. No rollback, no release.
