# Trust Center Blueprint

The public repository website should behave like a lightweight trust center.

## Trust center sections

- claim boundary;
- security posture;
- evidence standards;
- artifact schemas;
- release protocol;
- governance model;
- known non-goals;
- QA and digest reports;
- source lineage;
- contact and reporting path.

## Publication rule

The site should be generated from source-controlled content and QA reports, not hand-edited marketing pages. Public trust comes from repeatable publication gates.
