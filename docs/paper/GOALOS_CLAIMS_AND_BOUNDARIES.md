# GoalOS Canonical Claims and Boundaries

## Supported architectural claims

GoalOS defines a proof-to-capability operating architecture in which:

- objectives become bounded Proof Missions;
- unsupported claims become Proof Debt;
- Proof Debt becomes custom AGI Jobs;
- AGI Jobs return ProofBundles;
- Evidence Dockets make trust inspectable;
- Chronicle gates memory;
- admitted results become Validated Skills;
- Validated Skills form a governed graph;
- Merkle roots commit admitted state;
- encrypted capsules and zero-knowledge proofs can protect private intelligence;
- future missions may inherit only scoped, admitted capability.

## Current implementation claim

The project has a substantial production-grade specification and release-candidate implementation package: research papers, browser-local demonstrators, schemas, runbooks, reference Solidity contracts, GitHub Action packs, artifact exporters, and validation harnesses.

## Claims not currently made

The canonical release does not claim:

- achieved AGI or ASI;
- empirical state of the art;
- independent external validation;
- legal, security, compliance, or financial certification;
- guaranteed ROI or economic return;
- production mainnet deployment;
- live settlement in local mode;
- autonomous production authority;
- private plaintext secrecy on a public blockchain;
- measured compounding capability outside controlled or proxy environments.

## Conditions for stronger claims

Stronger claims require real tasks, controlled baselines, Evidence Dockets, replay logs, external reviewers, safety and cost ledgers, delayed outcomes, testnet or mainnet verification, incident response, rollback evidence, and independent reproduction.
