# GoalOS From Loop to RSI Governance Lab V1

**Tagline:** Build the governance institution before the system scales.

This browser-local public demo shows how a restartable GoalOS loop becomes a deterministic RSI governance cycle.

The public lesson:

```text
A prompt produces output.
A loop produces governed work.
A GoalOS RSI loop produces proof-gated recursive improvement.
```

## Core pipeline

```text
TARGET → EMIT → FILTER → ATLAS → TEST-PLAN → EVAL → INSERT → PROMOTE
```

## Public-alpha boundary

- No user data.
- No user funds.
- No wallet.
- No transaction.
- No network call.
- No production authority.
- Human review required.

## What the demo teaches

- OMNI-style interestingness may guide exploration allocation.
- OMNI must not have outcome authority.
- Confidence cannot inflate without evidence contact.
- High novelty raises skepticism.
- Move‑37 candidates require reproduction, stress testing, persistence, and dossier packaging.
- Recursive improvement must be gated by evidence, replay, baselines, risk, and human review.
