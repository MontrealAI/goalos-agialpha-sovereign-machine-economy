# GoalOS Institutional Deployment Wedge V1

**Start with one workflow. Earn the right to scale.**

This page is a browser-local public demo showing how a repeatable workflow moves through the GoalOS deployment wedge:

1. Choose repeatable workflow
2. Convert to GoalOSCommit
3. Execute bounded agents
4. Capture Evidence Docket
5. Run Selection Gate
6. Canary and monitor
7. Rollbackable expansion

The page is intentionally claim-bounded. It is a public-alpha demonstration, not production authorization, legal advice, investment advice, wallet support, or a transaction flow.

## User promise

- no user data
- no user funds
- no wallet
- no transaction
- no network call
- no production authority
- human review required

## Why it exists

The public demo suite already explains proof, dockets, ledgers, settlement, benchmarks, replay, and external review. This page answers the institutional adoption question: how does an organization begin?

The answer is one repeatable workflow and one proof-backed upgrade path.
