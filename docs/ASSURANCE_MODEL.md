# Assurance Model

The assurance model is built around evidence, reviewability, and bounded claims.

## Assurance controls

| Control | Purpose |
|---|---|
| Claim boundary | prevents unsupported public assertions |
| Mission contract | narrows ambiguous requests into auditable work |
| Tool trace | shows which actions were taken |
| Provenance record | records sources, artifacts, and dependencies |
| Validator report | records review conclusions and limitations |
| Risk ledger | preserves known risks and mitigations |
| Artifact digest | makes generated materials tamper-evident |
| Site QA | checks the generated public surface before publication |
| Release gate | separates demonstration from production authorization |

## Assurance principle

A system is not trusted because it sounds autonomous. It is trusted when its work can be inspected, replayed, challenged, bounded, and improved.
