# GoalOS Privacy Notice — Zero User Data Boundary

**Plain English:** We do not want your personal data. Do not submit personal data, customer data, confidential data, credentials, private keys, regulated records, payment data, health data, children’s data, trade secrets, or privileged information through the public website, GitHub issues, pull requests, demos, Evidence Dockets, or public proof runs.

## Public website posture

The GoalOS public website is intended to be static, no-account, no-tracking, no intentional analytics, no contact form, no payment form, no wallet connection, no model-call data collection, and no production action surface.

## GitHub interactions

If you interact with the repository through GitHub, GitHub processes that interaction under GitHub’s own terms and privacy notices. Public issues, comments, and pull requests may be public. Do not use them to submit personal, confidential, customer, or regulated data.

## Accidental submissions

If prohibited data is submitted, maintainers may remove, redact, close, or refuse it. Contributors remain responsible for what they submit.

## Pilots and private work

No private, enterprise, regulated, or customer-data processing is accepted through the public repository. Any such work requires a separate written agreement with appropriate privacy, confidentiality, security, retention, jurisdiction, and data-protection terms.

## GoalOS public-alpha boundary

GoalOS is a public-alpha proof architecture.

No user data. No user funds. GoalOS does not want user data. GoalOS does not want user funds. Do not submit personal, customer, confidential, regulated, credential, wallet, payment, private-key, seed-phrase, health, children’s, privileged, trade-secret, or proprietary data through public channels.
