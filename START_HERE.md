# Start Here — Super-Friendly GitHub Launch Guide ✨

You can launch this repository from GitHub.com without installing anything.

## What you are creating

You are creating a polished public repository and website for **proof-settled autonomous AI work**.

The action you paste into GitHub will create the repository files, generate the website, run quality checks, and publish GitHub Pages.

## The 7-click mental model

```text
Create repo
→ allow GitHub Actions to write
→ set Pages to GitHub Actions
→ paste one workflow file
→ run workflow
→ review reports
→ share website
```

## Exact path

1. Create a new GitHub repository named:

```text
goalos-agialpha-sovereign-machine-economy
```

2. Open:

```text
Settings → Actions → General → Workflow permissions
```

Choose:

```text
Read and write permissions
```

3. Open:

```text
Settings → Pages → Build and deployment → Source
```

Choose:

```text
GitHub Actions
```

4. Go back to **Code**.

5. Click:

```text
Add file → Create new file
```

6. Name the file exactly:

```text
.github/workflows/goalos-ascension-autopilot.yml
```

7. Paste the autonomous workflow and click **Commit changes**.

8. Open **Actions**.

9. Select:

```text
GoalOS AGIALPHA Ascension — Sovereign Machine Economy Autopilot
```

10. Click **Run workflow**.

Use:

```text
mode: direct_commit_and_publish
overwrite_existing_files: true
```

## What to check after it runs

Open these files first:

```text
public/index.html
reports/site-qa.json
reports/claim-scan.json
reports/repo-validation.json
reports/institutional-scorecard.json
reports/artifact-digests.json
```

Then open **Settings → Pages** and copy the live website URL into the repository **About** box.

## How to edit the website later

Edit only this file:

```text
content/site_manifest.json
```

Then rerun the same workflow.
