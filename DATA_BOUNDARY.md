# GoalOS Zero User Data Standard

## Absolute operating rule

GoalOS does not want user data. Do not submit it.

## Never submit

- Personal data.
- Customer, employee, patient, student, or user records.
- Confidential, proprietary, privileged, or trade-secret material.
- Credentials, API keys, tokens, passwords, seed phrases, private keys, wallet data.
- Payment, bank, tax, financial-account, health, biometric, children’s, criminal, government-ID, location, or regulated data.
- Non-public security vulnerabilities with exploit detail.

## Public proof / private intelligence

Public proof should use summaries, commitments, hashes, manifests, public sources, attestations, claim boundaries, and replay instructions that reveal no secrets. Private intelligence stays out of the public system.

## Violation response

Maintainers may remove, redact, close, or refuse content that appears to violate this policy. Submitters remain responsible for compliance and consequences.

## GoalOS public-alpha boundary

GoalOS is a public-alpha proof architecture.

No user data. No user funds. GoalOS does not want user data. GoalOS does not want user funds. Do not submit personal, customer, confidential, regulated, credential, wallet, payment, private-key, seed-phrase, health, children’s, privileged, trade-secret, or proprietary data through public channels.

The public website and demos are browser-local public demonstrations: no wallet, no transaction, no network call, no production authority, and human review required.

GoalOS does not claim achieved AGI, not achieved ASI, empirical SOTA, production authorization, legal approval, user-fund authorization, mainnet activation, not investment advice, trading advice, tax advice, or guaranteed outcomes.
