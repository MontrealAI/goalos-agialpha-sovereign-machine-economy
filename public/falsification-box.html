What would make
Proof Run 001 fail?

A serious proof system must publish its failure conditions before it claims success.

This page protects credibility. If the docket cannot be replayed, if claims are unsupported, if a cost/risk ledger is missing, or if the public/private boundary breaks, the proof run does not pass.

Failure conditions

  • No replay path.
  • No validator review.
  • Unsupported claims.
  • Missing cost or risk ledger.
  • Baseline comparison missing.
  • Unsafe or unsupported claim appears.
  • Result cannot be reproduced.
  • Docket is too vague for review.
  • Decision state is not inspectable.
  • Public proof leaks private intelligence.

Acceptance conditions

  • Mission Contract is clear.
  • Claims Matrix separates claimed and not claimed.
  • Evidence Docket exists.
  • Replay instructions are present.
  • Validator review is recorded.
  • Cost and risk ledgers are present.
  • Decision State is inspectable.
  • Human boundary remains active.
