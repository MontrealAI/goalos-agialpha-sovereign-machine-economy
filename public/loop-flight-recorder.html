02 / separate the roles

        
Role windows

        

          

            planner
            Creates contract
            
Turns the objective into success criteria, files, gates, and done conditions. It never writes final code or self-grades.

          

          

            generator
            Acts inside scope
            
Executes bounded work against the contract and writes artifacts to the disk-state pack.

          

          

            evaluator
            Pushes back
            
Reads traces, scores outcomes against the rubric, blocks unsupported claims, and requests restart when needed.

          

          

            harness
            Keeps loop minimal
            
Maintains just enough state, replay, and rollback to let the model drive without hiding failure.

     loop law
    
      
    Long-running autonomy is not a bigger prompt. It is a restartable institution.
    
      
    
        
    State lives on disk.Context windows lie, compact, and rot. Disk state makes restart, review, and replay possible.
    
        
    Roles stay separate.Planner, generator, evaluator, and harness do different work so the system does not grade itself.
    
        
    Traces are read.Every debugging insight starts with what the loop actually did, not what the prompt hoped it did.
    
        
    The bottleneck moves.When coding gets easier, verification, taste, restart, and evidence become the work.
