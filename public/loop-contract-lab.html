WHY THIS MATTERS

Prompting stops at output. Loops create institutions.

The GoalOS interpretation of long-running agent loops is simple: make the loop a governed object. Separate the planner, generator, and evaluator. Negotiate the contract before work begins. Write the state to disk. Let the loop restart. Read traces. Delete harness that the model no longer needs. Then make the next bottleneck visible.

GOALOS MAPPING

Loop → Contract → Trace → Docket → Capability.

A loop becomes GoalOS-native when it emits a Mission Contract, ProofPackets, trace summaries, an Evidence Docket plan, rollback / restart receipts, a Governed Decision State, and a capability package that can be reused only after review.
