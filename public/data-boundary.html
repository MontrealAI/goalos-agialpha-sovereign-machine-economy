Zero User Data Boundary

GoalOS uses public proof, not private intelligence. Do not submit personal, confidential, regulated, or customer data.

Data boundary: Public proof, private intelligence. GoalOS public pages and public dockets should contain only public-safe summaries, commitments, hashes, metadata, attestations, claim boundaries, and replay instructions. Private data stays out.

Public-safe

  • Mission title.
  • Public claims matrix.
  • Public sources.
  • Hash/pointer references.
  • Validator verdicts.
  • Cost/risk summaries.
  • Replay instructions that reveal no secrets.

Never public

  • Personal data.
  • Raw private prompts or traces.
  • Customer records.
  • Credentials and secrets.
  • Confidential tool outputs.
  • Privileged workpapers.
  • Sensitive security details.

Pilot rule

No pilot requester may submit private or regulated data through GitHub issues, public forms, public demos, public dockets, or public pull requests. Any private review requires a separate written agreement.

Proof without possession

The preferred pattern is to prove work using public-safe commitments, hashes, summaries, manifests, and attestations — not by collecting user data. GoalOS does not need your raw data to explain the public-alpha architecture.
