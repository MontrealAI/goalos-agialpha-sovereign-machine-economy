# Public-Safe RSI Governance Scenarios

These examples are safe for the browser-local demo and do not require user data, user funds, wallets, transactions, APIs, or private systems.

## Governed RSI pilot

Objective: Convert a restartable public demo loop into a deterministic RSI governance cycle with replay, baselines, ECI, and human review.

Expected state: `RSI_REVIEW_READY` only after hard gates pass.

## Move‑37 candidate handling

Objective: Evaluate a high-novelty candidate whose novelty and advantage exceed thresholds.

Expected state: `HOLD_MOVE37_DOSSIER_REQUIRED` until reproduction, stress testing, persistence, and dossier packaging pass.

## OMNI allocation stress

Objective: Show that interestingness can allocate exploration pressure but cannot authorize insertion, promotion, settlement, or deployment.

Expected state: `REJECT_OMNI_OUTCOME_AUTHORITY` if OMNI is given outcome authority.

## Sovereign dossier packaging

Objective: Package the candidate into a public-safe review dossier.

Expected state: `MOVE37_DOSSIER_REVIEW_READY` with human review still required.
