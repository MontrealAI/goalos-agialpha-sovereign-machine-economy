# Public-Safe Loop Scenarios

Use only public-safe objectives. Do not submit personal, customer, confidential, regulated, credential, wallet, private-key, seed-phrase, payment, trade-secret, proprietary, or user-fund data.

## Example scenarios

- Repository release hardening loop
- Public demo polish loop
- Proof mission loop
- Validator council loop

Each scenario should produce contract, state pack, trace ledger, restart check, bottleneck report, and reviewer brief.
