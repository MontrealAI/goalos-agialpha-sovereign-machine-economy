# Operator Console PRD

## Goal

Give non-technical operators a single surface for launching missions, reviewing evidence, checking validator reports, and approving release states.

## Core modules

- mission intake;
- job board;
- evidence docket viewer;
- validator review queue;
- risk ledger;
- release gate;
- Chronicle timeline;
- capability registry;
- public-site publication status.

## Success criteria

An operator can understand what happened, what remains uncertain, and what decision is allowed without reading raw code.
