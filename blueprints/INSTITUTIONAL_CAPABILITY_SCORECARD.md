# Institutional Capability Scorecard

Score the repository by operational readiness rather than speculative outcomes.

| Dimension | Target |
|---|---|
| Public clarity | non-technical users understand what the system is |
| Evidence discipline | claims map to proof artifacts |
| Operator control | pause, rollback, and release states are explicit |
| Governance | standards and exception paths are documented |
| Security posture | reporting and threat model exist |
| Automation | website and QA run through GitHub Actions |
| Reuse | Chronicle and Capability Packages preserve accepted work |
| Reviewability | schemas, examples, and reports are easy to inspect |
| Claim safety | unsupported claims are blocked or bounded |
| Extensibility | new missions, validators, and capabilities can be added cleanly |
